Ubuntu NVIDIA驱动完整部署指南

详细介绍在Ubuntu系统上部署NVIDIA驱动、CUDA和相关服务的完整指南，包括A800等高端GPU的配置
2025年08月24日 · 4 min · 720 words · Miao16

概述#

本文档提供了在Ubuntu系统上完整部署NVIDIA计算环境的详细指南，包括驱动安装、CUDA配置和相关服务设置。适用于深度学习、AI训练和高性能计算场景。

基础环境准备#

系统优化#

在开始安装之前，需要对系统进行基础优化：

锁定内核版本#

防止系统自动升级导致驱动不兼容：

1

apt-mark hold linux-generic linux-image-generic linux-headers-generic

关闭自动更新#

1
2

sudo sed -i.bak 's/1/0/' /etc/apt/apt.conf.d/10periodic &> /dev/null
sudo sed -i.bak 's/1/0/' /etc/apt/apt.conf.d/20auto-upgrades &> /dev/null

依赖安装#

安装编译必需的工具：

1

apt install gcc make -y

驱动下载#

NVIDIA Driver下载#

访问NVIDIA官方驱动下载页面： 官方链接: https://www.nvidia.cn/drivers/lookup/

根据您的GPU型号和操作系统版本选择合适的驱动版本。下载后会获得一个.run安装包，直接上传到服务器即可。

推荐版本：

  • 对于A800/A100系列：535.154.05或更高版本
  • 对于RTX系列：最新稳定版本

CUDA下载#

访问NVIDIA CUDA下载页面： 官方链接: https://developer.nvidia.com/cuda-downloads

选择对应的操作系统、架构和版本，推荐下载.run格式的安装包，便于自定义安装选项。

版本建议：

  • CUDA 12.2：适用于最新的深度学习框架
  • CUDA 11.8：兼容性较好，适用于较老的项目

NVIDIA Fabric Manager下载#

注意：仅SXM接口的GPU需要，PCIe接口的GPU不需要此组件。

1

wget https://developer.download.nvidia.cn/compute/cuda/repos/ubuntu2004/x86_64/nvidia-fabricmanager-535_535.154.05-1_amd64.deb

驱动安装#

NVIDIA Driver安装#

使用以下命令安装NVIDIA驱动（根据实际版本号修改）：

1
2
3
4
5
6

./NVIDIA-Linux-x86_64-535.154.05.run \
  --ui=none \              # 禁用图形界面（纯命令行模式）
  --no-questions \         # 跳过所有交互式提问（自动确认）
  --accept-license \       # 自动接受许可协议
  --disable-nouveau \      # 禁用开源驱动Nouveau
  --install-libglvnd       # 安装libglvnd兼容层

参数说明：

  • --ui=none：使用命令行界面，适合远程服务器
  • --no-questions：自动化安装，无需手动干预
  • --accept-license：自动接受NVIDIA许可协议
  • --disable-nouveau：禁用可能冲突的开源驱动
  • --install-libglvnd：安装OpenGL兼容层

CUDA安装#

安装CUDA Toolkit（根据实际版本号修改）：

1
2
3
4
5
6

./cuda_12.2.0_535.154.05_linux.run \
  --silent \               # 静默模式
  --toolkit \              # 仅安装CUDA Toolkit
  --override \             # 覆盖某些默认设置
  --no-drm \               # 不安装DRM库，避免安装GPU驱动程序
  --no-opengl-libs         # 不安装OpenGL库

重要提示：

  • 使用--no-drm和--no-opengl-libs避免CUDA安装程序覆盖已安装的驱动
  • --toolkit确保只安装CUDA工具包，不安装驱动程序

NVIDIA Fabric Manager安装#

仅适用于SXM接口的GPU（如A800、A100）：

1

dpkg -i nvidia-fabricmanager-535_535.154.05-1_amd64.deb

服务配置#

NVIDIA Fabric Manager服务#

启用和配置Fabric Manager服务（用于NVLink连接）：

1
2
3

systemctl daemon-reload
systemctl start nvidia-fabricmanager
systemctl enable nvidia-fabricmanager

NVIDIA Persistence Daemon#

创建并启用NVIDIA持久化守护程序：

 1
 2

cat << EOF > /lib/systemd/system/nvidia-persistenced.service
[Unit]
Description=NVIDIA Persistence Daemon
After=syslog.target

[Service]
Type=forking
PIDFile=/var/run/nvidia-persistenced/nvidia-persistenced.pid
Restart=always
ExecStart=/usr/bin/nvidia-persistenced --verbose
ExecStopPost=/bin/rm -rf /var/run/nvidia-persistenced/*
TimeoutSec=300

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable nvidia-persistenced
systemctl start nvidia-persistenced

作用说明：

  • 保持GPU驱动状态持久化
  • 避免GPU初始化延迟
  • 提高多用户环境下的GPU访问性能

环境配置#

CUDA环境设置#

将CUDA路径添加到系统环境变量：

1
2
3
4
5
6
7

cat << EOF >> /etc/profile
export PATH=/usr/local/cuda/bin:$PATH
export CUDA_HOME=/usr/local/cuda/
export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH
export LD_LIBRARY_PATH=/usr/local/cuda/targets/x86_64-linux/lib:$LD_LIBRARY_PATH
export LD_LIBRARY_PATH=/usr/local/cuda/targets/x86_64-linux/lib/stubs:$LD_LIBRARY_PATH
EOF

加载环境变量：

1

source /etc/profile

环境变量说明：

  • PATH：添加CUDA二进制文件路径
  • CUDA_HOME：CUDA安装根目录
  • LD_LIBRARY_PATH：CUDA库文件路径

常见问题解决#

驱动安装失败#

  1. Nouveau驱动冲突：

    1
2
3
4

    sudo bash -c "echo blacklist nouveau > /etc/modprobe.d/blacklist-nvidia-nouveau.conf"
sudo bash -c "echo options nouveau modeset=0 >> /etc/modprobe.d/blacklist-nvidia-nouveau.conf"
sudo update-initramfs -u
sudo reboot

  2. 内核版本不匹配：

    • 确保使用匹配的内核版本
    • 考虑降级或升级内核

GPU不可见#

  1. 检查PCIe连接：

    1

    lspci | grep -i nvidia

  2. 重新加载驱动：

    1
2

    sudo rmmod nvidia_uvm nvidia_drm nvidia_modeset nvidia
sudo modprobe nvidia

性能问题#

  1. 检查GPU时钟频率：

    1

    nvidia-smi -q -d CLOCK

  2. 设置性能模式：

    1
2

    sudo nvidia-smi -pm 1
sudo nvidia-smi -ac 1215,1410  # 根据GPU型号调整

最佳实践#

1. 版本兼容性#

  • 确保驱动、CUDA和深度学习框架版本兼容
  • 参考NVIDIA官方兼容性矩阵
  • 在生产环境中使用LTS版本

2. 系统维护#

  • 定期备份系统配置
  • 监控GPU温度和使用率
  • 建立驱动更新流程

3. 安全考虑#

  • 限制CUDA工具的访问权限
  • 定期更新安全补丁
  • 监控异常GPU活动
